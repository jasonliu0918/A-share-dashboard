## 1. 功能概览

一个用于盘中/盘后监控 A 股主要指数成交、市场广度、估值温度、市值结构、两融杠杆的轻量看板。**双击 HTML 即用**，无需后端。

### 1.1 顶部"指数汇总"区（最新布局）

整行结构：
- **第一行**：标题"指数汇总 · 上证 + 深证 + 北证50 · 交易日 YYYY-MM-DD"
- **第二行**：6 个等宽卡片
  1. **合计成交额（近3日）** — 沪+深+北证50 日线 K 求和，3 行（日期 / 柱 / 金额 / 日环比）
  2. **涨跌家数** — 上涨/下跌/平盘各占一行（红/绿/灰）
  3. **A股温度（自算）** — 全A PB 中位数 → 近10年分位 + ↻ 手动刷新按钮
  4. **集思录A股温度** — 读 `jsl_temperature.js`
  5. **A股总市值** — 上证A / 深证A / 北证A / 合计（万亿），时间在标题行
  6. **A股融资余额（近3日）** — 东财 datacenter RPTA_WEB_MARGIN_DAILYTRADE，3 行同款柱状

所有卡片高度通过 `align-items: stretch + height:100%` 拉齐统一。

### 1.2 指数卡片（默认 6 个，可勾选隐藏）

- 三大指数：上证(000001) / 深证(399001) / 北证50(899050)
- 扩展指数：沪深300(000300) / 创业板指(399006) / 科创50(000688)
- **每张卡片当前内容**：
  - 第一行：`指数名 + 代码`（左）｜ `涨跌家数：上涨/下跌/平盘`（右）
  - 最新价 + 涨跌额/涨跌幅（红绿配色）
  - 分时 mini SVG 走势图（可关）
  - 成交额（亿）— **成交量、峰值进度条、交易日标签均已移除**
- 卡片 padding 12/16，价格 28px，高度紧凑

### 1.3 分时 mini 图

- 300×72 SVG 折线图
- **横轴按真实时间线性映射**：`9:30→0`、`11:30→120`、`13:00→120`（午休折叠）、`15:00→240`
- 底部时间刻度 + 午休虚线 + 昨收水平虚线 + 涨跌色面积填充
- 独立刷新节奏（默认 30s），命中 `miniCache[code]` 直接重绘

### 1.4 涨跌家数

- 东财 `clist/get` 翻页（`pz=100`）拉各市场个股 `f3` + `f20`，本地统计 up/down/flat **以及总市值聚合**（同一请求复用，零额外开销）
- 市场 fs 映射：
  - 上证 `m:1+t:2,m:1+t:23` ｜ 深证 `m:0+t:6,m:0+t:80` ｜ 北证50 `m:0+t:81+s:2048`
  - 创业板 `m:0+t:80` ｜ 科创板 `m:1+t:23` ｜ 沪深300 不显示
- 20 秒 TTL 缓存
- 汇总仅合计三大指数所在市场

### 1.5 A股总市值（v7 新增）

- 复用涨跌家数翻页时一并累加 `f20`（总市值/元），单位转万亿
- 显示沪 / 深 / 北 + 合计
- 时间显示在卡片标题行

### 1.6 合计成交额近3日（v7 新增）

- 接口：`api/qt/stock/kline/get?klt=101&lmt=3`（日线 K）
- 同时拉上证/深证/北证50 三条日线，按日期合并求和
- 渲染同款柱状条（最新一日加重，前两日带日环比）
- 5 分钟自动刷新一次（受交易时段闸门约束，见 §1.10）

### 1.7 A股融资余额近3日

- 接口：`https://datacenter-web.eastmoney.com/api/data/v1/get?reportName=RPTA_WEB_MARGIN_DAILYTRADE&columns=STATISTICS_DATE,FIN_BALANCE,LOAN_BALANCE,FIN_BUY_AMT&pageSize=3`
- **此接口 JSONP 参数名是 `callback` 而非 `cb`**，单独实现
- 渲染同款柱状条

### 1.8 A股温度（两路独立口径）

**① 自算温度**（前端实时，命中即缓存，手动 ↻ 刷新）
- 东财 clist 翻页拉全 A 股 PB（`f23`）→ 过滤 PB≤0 → 求中位数
- 在 `pb_history.js` 近 10 年中位数排序数组里二分求分位
- 分档：<20 冰点 / <40 偏冷 / <60 温和 / <80 偏热 / ≥80 过热

**② 集思录 A股温度**（后台脚本生成，前端 `<script>` 引入）
- `gen_jsl_temperature.py` 裸 POST `https://www.jisilu.cn/data/indicator/get_last_indicator/`（**接口公开，不需要 cookie**）
- 写 `jsl_temperature.js`
- 由 Windows 任务计划 (schtasks) 定时运行

### 1.9 市场状态徽章

按本地时间 + `trade_date` 综合判定：盘中 / 今日收盘 / 盘前 / 休市·上一交易日。

### 1.10 自动刷新闸门（v7 新增）

`shouldAutoRefresh()` 闸门：
- **周末**（dow=0/6）：打开页面只跑一次 tick，**不自动续刷**
- **工作日**：09:25 之前 / 15:01 之后**停止自动刷新**
- 合计成交额 5 分钟轮询同样受闸门约束
- 闸门未排除节假日（如清明、五一），如需精确需补节假日表

### 1.11 设置面板（⚙）

| 配置项 | 可选值 | 默认 |
|---|---|---|
| 刷新间隔 | 1 / 3 / 5 / 10 / 30 秒 | 3 秒 |
| 数据源优先级 | 东财优先 → 腾讯兜底 / 反之 | 东财优先 |
| 显示指数 | 6 个复选框 | 全选 |
| 显示分时 mini 图 | 开关 | 开 |
| 显示"扩展指数"分组标题 | 开关 | 开 |
| 分时图刷新间隔 | 10–600 秒 | 30 秒 |

`localStorage['dashboard_cfg_v2']` 持久化。

## 2. 技术栈

- 前端：HTML + 原生 JS（ES2020）+ CSS Grid/Flex + SVG，无框架无构建；按样式、数据接口、状态逻辑、渲染、入口编排拆分
- 跨域：JSONP（行情、K 线、clist、datacenter-web 全部支持），CORS 不可绕的（PB 历史、集思录）改"后台脚本写静态 JS 文件"
- 数据脚本：Python 3 + `requests` + `akshare`，输出 `.js` 文件供 `<script>` 引入
- **Windows 上必须用 `py` launcher**（`python` 是 Store 占位 shim，exit 49）

容错：4 个东财域名轮询、双数据源主备切换、单请求超时 6–8s、部分失败可降级显示、各模块独立缓存 TTL。

### v6 → v7 的变化

1. **新增 3 个卡片**：A股总市值（沪/深/北 + 合计）、A股融资余额近3日、合计成交额近3日
2. **复用 clist 翻页**：把 `f20` 与 `f3` 合并到同一请求，零额外网络开销得到分市场总市值
3. **新增日线 K 接口调用** (`stock/kline/get?klt=101&lmt=3`) 实现合计成交额近3日
4. **新增 datacenter-web 接口调用**（融资余额），注意其 JSONP 参数名是 `callback`
5. **布局大改**：
   - 顶部"指数汇总"标题独占第一行（`grid-column: 1/-1`），含子标题与交易日
   - 6 个卡片均分第二行，统一高度
   - 涨跌家数改为"上涨/下跌/平盘"竖排 3 行
   - 各指数卡片：去掉成交量 metric、去掉蓝色峰值进度条、去掉交易日 badge；涨跌家数移到卡片右上角
   - 卡片 padding/字号整体压缩
6. **`shouldAutoRefresh()` 闸门**：周末完全静态、工作日 09:25–15:01 才自动续刷，避免无意义请求

## 7. 踩过的坑

1. bat 中文编码：GBK cmd 解析 UTF-8 bat 会拆烂命令，已弃用
2. akshare 不带 UA：`stock_zh_index_spot_em()` 被东财风控
3. `push2delay.eastmoney.com` 延迟略高但稳定，作为首选
4. 浏览器 CORS：东财/腾讯无 `Access-Control-Allow-Origin`，必须 JSONP
5. 原 `pulse` 闪烁动画用户不喜欢，改静默
6. sina 缺北证50 → 改腾讯
7. 分时图按点索引均分会拉伸，改为按真实分钟（0–240）映射
8. 午休折叠到同一 x，用虚线标注
9. `f104/f105/f106` 对**指数 secid** 是价位分布而非涨跌家数 → 必须 clist 翻页聚合
10. Windows `python` 是 Store shim → 用 `py` launcher
11. browser-cookie3 读 Chromium cookie 需管理员；后发现集思录接口本来就公开，整套 cookie 方案被移除
12. `file://` 下 `fetch` 本地 JSON 被拦 → 改 `<script>` 引入 `.js` 文件
13. **datacenter-web 的 JSONP 参数是 `callback` 而非 `cb`**，与 push2/clist 不一致
14. **kline 接口 `klines` 字段顺序是"日期,开,收,高,低,量(手),额(元),振幅,涨跌幅,涨跌额"** — 注意"成交额"在第 7 个（索引 6）
